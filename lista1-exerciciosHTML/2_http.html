Atividade 2

HTTP é a sigla para Hypertext Transfer Protocol, ou Protocolo de Transferência de Hipertexto o protocolo é utilizado para a tranferência de arquivos entre cliente e servidor na web por meio de requisições e respostas.

  1. Um navegador envia uma solicitação HTTP para a web
  2. Um servidor web recebe a solicitação
  3. O servidor executa um aplicativo para processar a solicitação
  4. O servidor retorna uma resposta HTTP (saída) para o navegador
  5. O navegador recebe a resposta
    Principais Métodos HTTP
  • GET
  • POST
  • PUT
  • HEAD
  • DELETE
  • PATCH
  • OPTIONS
  • CONNECT
  • TRACE
    Exemplo de Requisição Utilizando Método GET
  • GET /index.html HTTP/1.1
  • Host: www.exemplo.com
  • User-Agent: Google/5.0
    Cabeçalho de Requisição

    Um request HTTP contém vários campos que definem a ação que o cliente deseja realizar.

  • Content-Type: Define o tipo de dado enviado.
  • Authorization: Token para autenticação.
  • Accept: Tipo de resposta aceita.
  • User-Agent: Identifica o cliente.
  • Accept-Language: Preferência de idioma.
    Cabeçalho de Response

    A resposta HTTP traz a informação retornada pelo servidor após processar o request

  • Content-Type: Tipo do dado retornado
  • Cache-Control: Política de cache.
  • Set-Cookie: Define cookies.
  • ETag: Identificador para controle de cache.
